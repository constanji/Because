# 语义模型构建模板系统

## 概述

这是一个完整的语义模型构建模板系统，专为智能体（AI助手）设计，用于高效生成高质量的语义模型。系统包含8个核心模板，覆盖从规范定义到实现验证的完整流程。

## 目录结构

```
specs/semantic-model-templates/
├── semantic-model-spec-template.md          # 语义模型规范模板
├── semantic-model-plan-template.md          # 实现计划模板
├── semantic-model-tasks-template.md         # 任务清单模板
├── semantic-model-consistency-template.md   # 一致性分析模板
├── semantic-model-checklist-template.md     # 检查清单模板
├── semantic-model-constitution-template.md  # 项目原则模板
├── semantic-model-structure-template.md     # 文件结构模板
└── semantic-model-example-template.md       # 示例模板
```

## 模板说明

### 1. 语义模型规范模板 (`semantic-model-spec-template.md`)
用于定义语义模型的完整规范，包括：
- 模型目的和范围
- 核心概念和定义
- 关系结构
- 约束和规则
- 质量指标

### 2. 实现计划模板 (`semantic-model-plan-template.md`)
用于制定语义模型的实现计划，包括：
- 实施策略
- 阶段划分
- 资源分配
- 风险评估
- 时间线

### 3. 任务清单模板 (`semantic-model-tasks-template.md`)
用于生成具体的实施任务清单，包括：
- 任务分解
- 优先级排序
- 依赖关系
- 验收标准
- 负责人分配

### 4. 一致性分析模板 (`semantic-model-consistency-template.md`)
用于分析语义模型的一致性，包括：
- 概念一致性检查
- 关系完整性验证
- 约束冲突检测
- 命名规范检查
- 质量评估

### 5. 检查清单模板 (`semantic-model-checklist-template.md`)
用于创建语义模型的质量检查清单，包括：
- 设计原则检查
- 实现质量检查
- 文档完整性检查
- 测试覆盖检查
- 部署准备检查

### 6. 项目原则模板 (`semantic-model-constitution-template.md`)
用于定义语义模型项目的核心原则，包括：
- 设计哲学
- 质量标准
- 团队规范
- 变更管理
- 持续改进

### 7. 文件结构模板 (`semantic-model-structure-template.md`)
用于定义语义模型的文件结构，包括：
- 目录组织
- 文件命名规范
- 模块划分
- 接口定义
- 文档结构

### 8. 示例模板 (`semantic-model-example-template.md`)
提供完整的语义模型示例，包括：
- 完整规范示例
- 实现示例
- 测试示例
- 文档示例
- 部署示例

## 使用说明

### 快速开始

1. **创建语义模型规范**：
   ```bash
   # 使用语义模型规范模板
   speckit specify --template semantic-model-spec-template.md "创建一个用户画像语义模型"
   ```

2. **生成实现计划**：
   ```bash
   # 使用实现计划模板
   speckit plan --template semantic-model-plan-template.md "基于用户画像规范制定实施计划"
   ```

3. **生成任务清单**：
   ```bash
   # 使用任务清单模板
   speckit tasks --template semantic-model-tasks-template.md "分解用户画像实现任务"
   ```

### 模板使用流程

1. **需求分析**：使用规范模板定义语义模型需求
2. **计划制定**：使用计划模板制定实施策略
3. **任务分解**：使用任务模板生成具体任务
4. **一致性检查**：使用分析模板验证模型一致性
5. **质量保证**：使用检查清单模板确保质量
6. **原则定义**：使用原则模板建立项目标准
7. **结构设计**：使用结构模板定义文件组织
8. **示例参考**：使用示例模板获取最佳实践

## 占位符系统

### 动态占位符（运行时替换）
- `$ARGUMENTS`：用户输入参数
- `{ARGS}`：上下文参数
- `[###-model-name]`：模型分支名称模式
- `[日期]`：当前日期
- `[模型名称]`：语义模型名称

### 静态占位符（模板填充时替换）
- `[占位符-描述]`：需要根据上下文填充的内容
- `[选项 1/选项 2]`：多选占位符
- `[概念名称]`：语义模型中的概念名称
- `[关系类型]`：概念之间的关系类型
- `[约束条件]`：模型的约束条件

## 最佳实践

### 1. 语义模型设计
- **概念清晰**：每个概念应该有明确的定义和边界
- **关系明确**：关系应该具有明确的语义和方向
- **约束合理**：约束应该确保模型的完整性和一致性
- **层次适当**：概念层次应该反映实际领域结构

### 2. 模板使用
- **顺序执行**：按照模板顺序逐步构建语义模型
- **迭代优化**：多次迭代优化模型质量
- **一致性检查**：每个阶段都进行一致性验证
- **文档完整**：确保所有模板都完整填写

### 3. 质量保证
- **概念验证**：验证每个概念的实用性和必要性
- **关系验证**：验证关系的正确性和完整性
- **约束验证**：验证约束的一致性和可行性
- **性能评估**：评估模型的查询和推理性能
